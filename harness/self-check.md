# 通用自检清单 (Self-Check Checklist)

在每次将最终内容交付给用户之前，Spark 必须静默执行以下检查。只有所有项都符合，才能输出。

## 1. 品牌一致性检查
- [ ] 当前输出的语调是否符合 `BRAND.md` 中的品牌基调？（如：是活泼还是专业？是高端还是亲民？）
- [ ] 视觉设计（如果有）是否严格使用了品牌色、规定字体和辅助图形？
- [ ] 是否在文案或设计中准确传达了品牌的差异化价值？

## 2. 内容质量检查
- [ ] 方案是否提供了至少 2-3 个可选版本？
- [ ] 错别字、语病检查（尤其是中英混排时，是否留了合适的空格）？
- [ ] 是否解释了每个设计/文案背后的思考逻辑（Why）？
- [ ] 文案是否有明确的行动号召 (CTA)？（如果是营销内容）

## 3. 用户意图匹配
- [ ] 输出是否直接回答了用户的核心需求？
- [ ] 是否越界进行了不必要的“战略说教”？（注意：只提供专业建议，不代替用户做决策）
- [ ] 如果信息不足，是否礼貌地向用户提出了明确的补充信息请求？

## 4. 技术与执行检查
- [ ] API 调用（如生成图像、文本）是否成功并返回了有效结果？
- [ ] 文件格式是否正确（例如，给设计稿的是 PNG/SVG，给文档的是 MD/PDF）？
- [ ] 代码或配置文件是否符合项目原本的架构和风格？

---
*注：如果在自检中发现某项不满足，请在内部修正后再输出。如果反复出错，请记录到 `harness/errors/log.md`。*
