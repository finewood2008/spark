# 火花 (Spark) - 商业级 AI 营销工作台 PRD

> **文档目的**：本 PRD 用于指导后续 AI 智能体（或人类开发者）全面理解火花 (Spark) 的产品定位、架构逻辑、功能模块与后续演进方向，确保多方协作时的认知对齐。

## 1. 产品定位与心智
**火花 (Spark)** 不是一个“通用大模型聊天窗口”，而是一个**专属的数字 CMO（首席营销官）与品牌设计工作台**。
它的核心使命是帮助**中小企业/个体户**建立、管理和传播其一致的品牌视觉与营销内容。
它的心智模型是**“左脑沟通决策 + 右脑无限画板”**的桌面办公软件。

## 2. 核心架构与技术栈
* **宿主环境**：Electron 桌面端应用（支持 macOS / Windows）。
* **UI 层**：React 18 + Tailwind CSS + Vite。
* **桌面特性**：依赖本地文件系统（读取/写入本地文件）、本地 IPC 进程通信。
* **数据/模型层**（规划中）：Python 后端（用于对接本地或云端的视觉模型如 SAM, Stable Diffusion 等）以及通过 LLM API 进行文本生成和 RAG 知识管理。

## 3. UI 布局与交互范式 (双栏设计)
火花采用了业界前沿的 **Agentic UI 范式**：
* **最左侧**：全局导航侧边栏（Icon-based）。
* **左侧中栏 (Chat)**：固定宽度（如 450px），属于用户的**沟通指令区**。这里运行着一个名叫 **Alex** 的专属 CMO 数字人。所有的工作分配、策略沟通、修改意见都在这里以自然语言形式进行。
* **右侧主栏 (Workspace)**：动态扩展，属于**工作台与结果呈现区**。当左侧 Alex 生成了成果物时，右侧会根据不同的任务类型展示相应的可视化工作台。

## 4. 核心功能模块详细说明

### 4.1 沟通区 (Chat) - `src/frontend/renderer/pages/Chat/index.tsx`
* **破冰与预设动作 (Onboarding)**：首次进入时，提供快速指令按钮（如“写篇初夏推文”），降低新用户写 prompt 的门槛。
* **多模态输入**：支持文本、上传参考图（将转为指令加入输入框）和调用本地品牌知识库。
* **结果反馈卡片**：Agent 生成内容后，在聊天流中以“卡片”形式呈现摘要。
* **版本穿梭机 (Time Machine)**：卡片右上角自带版本切换器（如 `< v1/2 >`），允许用户在同一次任务的不同修改版本间快速无缝切换，避免翻看长聊天记录。

### 4.2 无限画板工作台 (Canvas Workspace) - `src/frontend/renderer/components/CanvasWorkspace.tsx`
* **触发场景**：当大模型生成了图片或长文本时。
* **交互逻辑**：采用类似 Figma / Miro 的无限画布设计。背景为点阵网格，支持 `Ctrl+滚轮` 缩放，鼠标中键/右键拖拽平移。
* **原子化内容实体**：生成的图文不再是呆板的只读页面，而是化作画板上的可拖拽“便利贴卡片”。多个生成的卡片可以并排对比。
* **本地与大模型联动**：
  * 支持通过 Electron 的系统原生对话框（`dialog:openImage`）直接导入用户本地硬盘的图片到画板上。
  * **悬浮工具栏与 AI 魔法**：鼠标悬浮在图片上，可以直接唤出“抠图”、“扩图”、“AI 魔法（自然语言修图）”等基于视觉大模型的操作入口。操作时会生成“草稿占位卡片”，异步等待模型返回结果后更新。

### 4.3 品牌矩阵与 VI 超市 (Brand Center) - `src/frontend/renderer/pages/BrandCenter.tsx`
* **设计意图**：实现商业化闭环与多实体管理。
* **我的品牌实体 (My Brands)**：用户可以建立多个不同的商业品牌（如 A 咖啡馆，B 服装店）。每个品牌是一个隔离的 Sandbox，包含它自己独立的 RAG 知识库、主色调、调性描述和生成的视觉资产。用户可**一键激活 (Switch)** 某品牌，后续的所有 Agent 对话和生成都会受制于该品牌的全局设定。
* **VI 方案超市 (Market)**：类似游戏皮肤商店。展示预设的高级 VI 视觉方案（包括 Logo、标准色、标准字体和版式排版方案），明码标价。用户点击购买后，该方案会瞬间转化为用户的一个“品牌实体”，且自带大模型的二次创作能力（如：基于购买的极简风方案，让大模型把名字换成自己的店名）。

### 4.4 其他辅助模块（持续迭代中）
* **知识库 (Knowledge Base)**：管理品牌文档与规则，供 RAG 引擎调用。
* **发布管道 (Publish)**：连接各大社交平台（微信、小红书等）的一键分发中心。

## 5. 后续 AI 开发者接手开发指南
如果你是一个接手该项目的 AI 智能体，请遵循以下原则：
1. **优先组件化**：所有的右侧工作台模块均作为独立的组件渲染，状态应由全局（或最顶层 App）管理。
2. **模拟后端先行**：在没有真实接入 Python CV 模型和复杂 LLM RAG 前，所有涉及到“生成”、“抠图”的操作请继续使用 `setTimeout` 和状态更新来模拟交互链路，确保前端 UI 流畅跑通。
3. **不要破坏 Electron IPC 机制**：前端 (`renderer`) 严禁直接使用 Node.js API，必须通过 `preload.ts` 中暴露的 `window.spark` 方法向 `main.ts` 发起请求，确保安全性。
4. **视觉一致性**：新页面的色彩需遵循主品牌色 `#FF6B35` (火焰橙) 与圆角设计规范（如 `rounded-2xl`，大面积留白，轻阴影 `shadow-sm`）。

## 6. 近期 Roadmap 建议
1. **打通真实的本地大模型/API**：将 Chat 接口替换为真实的 OpenAI/Claude 接口。
2. **连接本地图像处理**：将无限画布的抠图指令发送给本地部署的 Python 脚本 (如 `rembg` 或 SAM 模型)。
3. **完善图文排版编辑器**：在画板上支持真正的图文图层合并导出。
