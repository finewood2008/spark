# SPEC.md - 火花产品规格书

> 火花 (Spark) v1.0 产品规格文档

---

## 一、产品概述

### 1.1 产品定位
**一句话**：企业的新媒体部门，用 AI 驱动。

### 1.2 核心问题
中小企业需要内容营销，但没有预算雇佣专业团队。火花让一个人也能完成一个部门的工作。

### 1.3 解决方案
通过 AI 技术栈（知识库 + 生成 + 发布），实现从"内容构思"到"多平台分发"的全流程自动化。

## 二、用户画像

### 2.1 主要用户

| 用户类型 | 特征 | 需求 |
|---------|------|------|
| **小微企业主** | 老板自己兼运营 | 快速出内容，不耽误主业 |
| **市场部负责人** | 1-3人团队 | 提效+保质+多平台 |
| **创业团队** | 无专职工新媒体 | 冷启动，低成本 |
| **传统企业** | 转型数字化 | 需要系统化内容管理 |

### 2.2 用户核心旅程
```
认知 → 试用 → 冷启动配置 → 日常使用 → 依赖 → 传播
```

## 三、功能模块详细设计

### 3.1 模块一：企业 AI 大脑（知识库系统）

#### 3.1.1 知识输入

| 输入方式 | 支持格式 | 处理方式 |
|---------|---------|---------|
| **对话引导** | 实时对话 | AI 主动提问，引导用户填写 |
| **文件上传** | PDF/Word/PPT/Excel/ TXT/MD | 自动解析提取关键信息 |
| **图片上传** | PNG/JPG/SVG | OCR 识别 + 视觉分析 |
| **链接抓取** | URL | 爬取网页内容摘要 |
| **音频上传** | MP3/WAV | Whisper 转写后分析 |

#### 3.1.2 知识库结构

```
企业知识库
├── 企业画像
│   ├── 基本信息（名称、行业、规模、地区）
│   ├── 使命/愿景/价值观
│   ├── 核心竞争优势
│   └── 目标客群画像
├── 产品知识
│   ├── 产品列表
│   ├── 产品功能/卖点
│   ├── 定价策略
│   └── FAQ
├── 品牌资产
│   ├── Logo 文件
│   ├── VI 规范（配色/字体/图形）
│   └── 品牌故事
└── 运营数据
    ├── 历史高表现内容
    └── 用户反馈
```

#### 3.1.3 知识库技术实现
- **向量存储**：ChromaDB / Qdrant
- **全文检索**：BM25 + 向量混合搜索
- **文件处理**：Unstructured.io + Tika
- **图片理解**：Claude Vision / GPT-4V

---

### 3.2 模块二：VI 智能系统

#### 3.2.1 VI 现有检测
- 自动检测用户是否已有 VI（上传 Logo 等）
- 已有时：规范化整理入库

#### 3.2.2 VI 自动生成

**触发条件**：用户无 VI 或主动申请生成

**生成流程**：
```
1. 采集企业信息
   - 行业属性
   - 品牌关键词（用户选择或 AI 推荐）
   - 目标客群风格偏好
   
2. AI 生成方案（3 套供选）
   - 方案A：专业稳重型
   - 方案B：活力年轻型
   - 方案C：高端精致型
   
3. 用户选择 + 微调
   - 调整主色调
   - 换字体
   - 修改 Logo 概念
   
4. 确认后输出
   - Logo 文件（SVG/PNG）
   - VI 手册（PDF）
   - 字体文件
   - 配色色值
```

**技术工具**：
- Logo：DALL-E 3 / Midjourney 生成概念 + Adobe Illustrator 矢量化
- 配色：AI 基于色彩理论生成
- 字体：Google Fonts / 思源字体

### 3.3 模块三：内容生产中心

#### 3.3.1 支持平台

| 平台 | 内容形式 | 特殊要求 |
|------|---------|---------|
| 微信公众号 | 图文文章 | 排版、封面图、摘要 |
| 小红书 | 种草笔记/图文帖 | 话题标签、emoji |
| 抖音/视频号 | 短视频脚本+封面 | 时长、字幕 |
| 微博 | 短图文/话题 | 字数限制、话题 |
| 知乎 | 长文/回答 | 专业性、引用 |
| B站 | 视频脚本/图文 | 标题党、封面 |
| 官网/电商 | 产品描述 | SEO、转化导向 |
| 邮件/短信 | 营销文案 | 短小精悍 |

#### 3.3.2 内容生成流程
```
用户输入
  ↓
解析意图（主题/关键词/平台/风格）
  ↓
RAG 检索（从知识库提取相关内容）
  ↓
Prompt 组装（注入品牌VI+产品信息+风格）
  ↓
AI 生成（3 套方案）
  ↓
自动注入 VI 元素（配色块/字体/Logo水印）
  ↓
预览 + 用户修改
  ↓
确认发布 / 定时发布
```

#### 3.3.3 内容一致性保证
- **品牌调性**：每次生成前加载品牌指南
- **VI 一致**：自动在图片/排版中应用品牌色
- **产品一致**：从产品知识库提取，避免信息错误
- **风格记忆**：记录用户修改偏好，下次优化

### 3.4 模块四：多平台发布

#### 3.4.1 支持平台及接入方式

| 平台 | 接入方式 | 能力 |
|------|---------|------|
| 微信公众号 | 官方 API | 发布、预览、定时 |
| 小红书 | Cookie 模拟 | 发布、话题 |
| 抖音 | 官方开放平台 | 视频上传、字幕 |
| 视频号 | 官方 API | 发布 |
| 微博 | 官方 API | 发布 |
| 知乎 | OAuth | 发布 |
| B站 | 官方 API | 发布 |

#### 3.4.2 发布调度
- **即时发布**：一键发送到所有选中平台
- **定时发布**：内容日历 + 定时任务
- **队列管理**：发布失败自动重试

#### 3.4.3 数据回传
- 发布成功后记录发布状态
- 追踪阅读量/点赞/评论（定期轮询）
- 高表现内容自动标记为优质案例

### 3.5 模块五：自我进化系统

#### 3.5.1 学习机制

| 数据来源 | 学习内容 |
|---------|---------|
| 用户修改 | 内容风格偏好、常用词汇 |
| 用户反馈 | 点赞/不满意的点 |

#### 3.5.2 进化表现
- 生成内容越来越符合用户品牌调性
- 常用模板一键调用
- 智能推荐"你应该发这个"

### 4.2 前端技术栈
- **框架**：Electron + React + TypeScript
- **状态管理**：Zustand
- **UI 组件**：Radix UI / shadcn/ui
- **样式**：Tailwind CSS
- **本地通信**：IPC + WebSocket

### 4.3 后端技术栈
- **运行时**：Node.js 20+
- **Agent 框架**：LangGraph / 自研 FSM
- **向量数据库**：ChromaDB（本地）
- **文件处理**：Unstructured.io, Tika, Whisper
- **AI 提供商**：vveai API（Claude/GPT/Gemini）

### 4.4 数据存储
- **向量数据**：ChromaDB（本地文件）
- **结构化数据**：SQLite（本地）
- **文件资产**：本地文件系统 + 云同步（可选）
- **配置信息**：JSON 文件
